## 🔍 요약

- `gap`은 Flexbox, Grid에서 요소 간의 간격을 정의하는 속성
- 이제는 `gap` 공간에 선, 색상, 스타일을 **적용 가능**
- `column-rule`, `row-rule` 같은 **새로운 속성 등장**
- 그러나 **실제 렌더링은 아직 실험적 단계**이며 **기본 브라우저에서는 보이지 않음**
- ⚠️ `column-rule`은 기존 multi-column 레이아웃에서도 쓰였던 속성이므로 **혼동 주의!**
- 🧪 테스트는 **[playground](https://microsoftedge.github.io/Demos/css-gap-decorations/playground.html)** 에서!

---

## 🆕 새로운 속성 소개

| 속성명 | 설명 | 지원 상태 (2025년 7월 기준) |
| --- | --- | --- |
| `column-rule` | **Flex/Grid의 `gap` 영역에 선을 그리는 용도** | ⚠️ Chrome 139+에서 실험 플래그 필요 |
| `row-rule` | 수평 간격(행 간격)에 스타일 적용 | ❌ 미지원 |
| `gap-style`, `gap-color` 등 | 향후 `gap`에 직접 스타일 지정 가능성 (제안 중) | ⏳ 논의 중 |

> 📌 **주의**: `column-rule`은 **기존 multi-column 레이아웃에서도 사용되는 속성**입니다.
Flex/Grid에서 gap에 적용하는 `column-rule`은 **완전히 다른 용도**입니다.
🔗 [MDN 문서 (multi-column용)](https://developer.mozilla.org/en-US/docs/Web/CSS/column-rule)
>

## 📸 실험 예시: Flex에 스타일이 있는 `gap` 적용 – *해봤는데 안되더라*

```css
.container {
  display: flex;
  flex-direction: row;
  gap: 20px;
  column-rule: 10px dotted rebeccapurple;
  background: #f9f9f9;
  padding: 16px;
  align-items: stretch;
}
```

<!-- ![image.png](image.png) -->

> 📌 Chrome 138에서 `column-rule`을 적용했지만 **실제 화면에서는 선이 렌더링되지 않음.**
개발자 도구 상에서는 속성이 적용된 것으로 보이지만 **시각적으로는 무효.

Chrome/Edge 139 이상에서 `#enable-gap-decoration` 실험 플래그를 켜야만 렌더링**됩니다. 
(주의 사항을 꼭 읽어봅시다…^^)
또는 [**playground](https://microsoftedge.github.io/Demos/css-gap-decorations/playground.html)** 에서 테스트 가능합니다.

## 📦 브라우저 지원 현황

| 브라우저 | `column-rule` (gap용) | `row-rule` |
| --- | --- | --- |
| Chrome 126~138 | ⚠️ CSS 속성만 인식됨 (렌더링 안 됨) |  |
| Chrome 139+ | ✅ **실험 플래그 활성화 시** 렌더링 |  |
| Firefox Nightly | 🧪 일부 실험적 구현 |  |
| Safari | ❌ 미지원 |  |

---

## 🔧 실무 적용 시 유의사항

- `column-rule` (gap용)은 **현재 일반 브라우저에서는 작동하지 않음**
- 반드시 `#enable-gap-decoration` 실험 기능을 켜야 작동
- 실무에서는 `::before`, `border-right`, `outline` 등으로 대체 구현 필요
- `@supports (column-rule: solid)`는 true이지만 **시각적으로는 작동 안 함**
- `column-rule`이라는 이름 때문에 **multi-column과 혼동하지 않도록 주의 필요**

## 🔮 향후 전망

- `gap-style`, `gap-color`, `gap-border` 등의 확장이 논의 중
- 레이아웃의 **논리적 구조와 시각적 표현을 분리**하면서도 유연한 스타일링이 가능해질 것으로 기대
- 렌더링 엔진이 안정화되면 더욱 직관적인 디자인 구현 가능
- 디자이너와 개발자 간의 협업 흐름 개선
- 크로스브라우징 & 접근성 측면에서 **지속 관찰 필요**
