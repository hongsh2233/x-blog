## 서버 컴포넌트란?

**React Server Components(RSC)**는 서버에서 렌더링되며 클라이언트로 HTML 결과만 전송하는 컴포넌트입니다. JavaScript 번들에 포함되지 않기 때문에 번들 크기를 크게 줄일 수 있습니다.

## 서버 vs 클라이언트 컴포넌트 비교

| 특성 | 서버 컴포넌트 | 클라이언트 컴포넌트 |
|------|------------|----------------|
| 실행 환경 | 서버 | 브라우저 |
| useState / useEffect | ❌ 사용 불가 | ✅ 사용 가능 |
| DB / 파일시스템 접근 | ✅ 직접 접근 | ❌ API 필요 |
| 번들 크기 | 포함 안 됨 | 포함됨 |
| 인터랙션 | ❌ 없음 | ✅ 이벤트 처리 |

## 언제 서버 컴포넌트를 써야 하나?

```tsx
// ✅ 좋은 사용 예: 데이터 페칭이 필요한 컴포넌트
// app/posts/page.tsx (서버 컴포넌트 - 기본값)
async function PostsPage() {
  // 직접 DB 또는 파일시스템 접근 가능
  const posts = await getAllPosts();

  return (
    <div>
      {posts.map(post => (
        // 인터랙션 없는 순수 표시용 컴포넌트
        <PostCard key={post.slug} post={post} />
      ))}
    </div>
  );
}

// ✅ 좋은 사용 예: 정적 레이아웃
function BlogLayout({ children }: { children: React.ReactNode }) {
  return (
    <div className="max-w-4xl mx-auto">
      <nav>네비게이션 (서버에서 렌더링)</nav>
      <main>{children}</main>
    </div>
  );
}
```

## 클라이언트 컴포넌트는 언제?

```tsx
// ✅ 클라이언트 컴포넌트가 필요한 경우
'use client';

import { useState, useEffect } from 'react';

function LikeButton({ slug }: { slug: string }) {
  const [likes, setLikes] = useState(0);
  const [isLiked, setIsLiked] = useState(false);

  useEffect(() => {
    // 브라우저 이벤트, localStorage, API 호출 등
    fetch(`/api/posts/${slug}/likes`)
      .then(r => r.json())
      .then(data => setLikes(data.likes));
  }, [slug]);

  const handleLike = async () => {
    setLikes(l => l + 1); // 낙관적 업데이트
    setIsLiked(true);
    await fetch(`/api/posts/${slug}/likes`, { method: 'POST' });
  };

  return (
    <button onClick={handleLike}>
      {isLiked ? '❤️' : '🤍'} {likes}
    </button>
  );
}
```

## 컴포넌트 경계 설계 전략

서버/클라이언트 컴포넌트를 적절히 혼합하는 것이 핵심입니다.

```
PostDetailPage (서버)
├── PostHeader (서버) ← 정적 메타데이터
├── PostContent (서버) ← 마크다운 HTML 렌더링
└── PostInteractions (클라이언트) ← 좋아요, 댓글 등
    ├── LikeButton (클라이언트)
    └── ViewCounter (클라이언트)
```

"클라이언트 컴포넌트는 트리의 말단(leaf)으로 밀어내라"는 원칙을 기억하세요. 서버 컴포넌트가 상위에 있고, 상호작용이 필요한 부분만 클라이언트 컴포넌트로 분리하면 최적의 성능을 얻을 수 있습니다.

## 정리

RSC는 은탄환이 아닙니다. 데이터 페칭, 정적 렌더링에는 서버 컴포넌트를, 사용자 인터랙션과 브라우저 API 사용에는 클라이언트 컴포넌트를 적재적소에 활용하세요. Next.js App Router에서는 기본이 서버 컴포넌트이므로, `'use client'`가 필요한 경우에만 선언하는 습관을 들이는 것이 좋습니다.
