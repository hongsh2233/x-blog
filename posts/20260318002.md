## 왜 제네릭인가?

타입을 함수의 인자처럼 받아서 재사용 가능한 컴포넌트를 만드는 것이 **제네릭(Generics)**의 핵심입니다. 중복 코드를 없애고 타입 안전성을 유지할 수 있습니다.

```typescript
// 제네릭 없이 - 중복 발생
function getFirstString(arr: string[]): string {
  return arr[0];
}
function getFirstNumber(arr: number[]): number {
  return arr[0];
}

// 제네릭 사용 - 하나로 통합
function getFirst<T>(arr: T[]): T {
  return arr[0];
}

const firstStr = getFirst<string>(['hello', 'world']); // string
const firstNum = getFirst<number>([1, 2, 3]);           // number
```

## 유용한 유틸리티 타입들

TypeScript 내장 유틸리티 타입은 기존 타입을 변환하여 새로운 타입을 만들어 줍니다.

```typescript
interface User {
  id: number;
  name: string;
  email: string;
  password: string;
  createdAt: Date;
}

// Partial<T> - 모든 필드를 선택적으로
type UserUpdate = Partial<User>;
// { id?: number; name?: string; email?: string; ... }

// Pick<T, K> - 특정 필드만 선택
type UserProfile = Pick<User, 'id' | 'name' | 'email'>;
// { id: number; name: string; email: string; }

// Omit<T, K> - 특정 필드 제외
type UserPublic = Omit<User, 'password'>;
// { id: number; name: string; email: string; createdAt: Date; }

// Required<T> - 모든 필드를 필수로
type StrictUser = Required<UserUpdate>;
```

## 조건부 타입 (Conditional Types)

```typescript
type IsArray<T> = T extends any[] ? true : false;

type A = IsArray<string[]>;  // true
type B = IsArray<string>;    // false

// infer 키워드로 타입 추출
type UnpackArray<T> = T extends (infer U)[] ? U : T;

type C = UnpackArray<string[]>;  // string
type D = UnpackArray<number>;    // number
```

## 실전 패턴: API 응답 타입 관리

```typescript
// 공통 API 응답 래퍼
interface ApiResponse<T> {
  data: T;
  status: number;
  message: string;
  timestamp: string;
}

// 특정 엔드포인트 응답 타입
interface Post {
  id: string;
  title: string;
  content: string;
  author: Pick<User, 'id' | 'name'>;
}

type PostListResponse = ApiResponse<Post[]>;
type PostDetailResponse = ApiResponse<Post>;

// 타입 가드
function isPost(data: unknown): data is Post {
  return (
    typeof data === 'object' &&
    data !== null &&
    'id' in data &&
    'title' in data
  );
}
```

## 정리

제네릭과 유틸리티 타입을 잘 활용하면 반복적인 타입 정의를 줄이고, 더 안전하고 유지보수하기 좋은 코드를 작성할 수 있습니다. `Partial`, `Pick`, `Omit`, `Required`는 특히 API 연동 시 자주 사용되므로 반드시 익혀두세요.
