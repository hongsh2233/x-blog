## 🔧 제미나이 CLI란?

구글이 발표한 **오픈소스 AI 에이전트 도구**로, 터미널에서 Gemini 모델을 활용할 수 있는 CLI(Command Line Interface)입니다.

- GPT와 유사한 LLM 기반 AI 모델을 CLI에서 바로 활용
- 명령어 입력만으로 코드 분석, 리팩토링, 설명, 생성 가능
- Google Cloud, VS Code, Cloud Code 등과 연동 지원

## 🛠️ 사용 방법

1. 설치

```bash
bash
npm install -g @google/gemini-cli
```

## 🧠 Gemini CLI 명령어 정리 (GitHub 공식 문서 기준)

### ✅ Slash (`/`) 명령어

🔹 **Slash 명령어 (`/`)**

| 명령어 | 설명 |
| --- | --- |
| `/about` | 현재 Gemini CLI의 버전과 시스템 정보를 표시합니다. |
| `/auth` | 인증 방식을 변경합니다. |
| `/bug <title>` | 문제를 보고하고 GitHub 저장소에 이슈를 생성합니다. |
| `/chat save <tag>` | 현재 대화를 태그와 함께 저장합니다. |
| `/chat resume <tag>` | 저장된 태그를 불러와 대화를 재개합니다. |
| `/chat list` | 저장된 모든 태그 목록을 표시합니다. |
| `/clear` | 현재 터미널 창을 지웁니다. (`Ctrl+L` 단축키로도 가능) |
| `/compress` | 현재 대화를 압축하여 컨텍스트를 간결하게 유지합니다. |
| `/editor` | 사용할 편집기를 선택합니다. |
| `/extensions` | 현재 활성화된 확장 기능 목록을 표시합니다. |
| `/help` 또는 `/?` | 명령어 도움말을 표시합니다. |
| `/mcp desc` | 현재 설정된 MCP 서버의 설명을 표시합니다. |
| `/mcp schema` | MCP 서버에서 수신한 스키마를 표시합니다. |
| `/memory add <내용>` | 메모리(GEMINI.md)에 항목을 추가합니다. |
| `/memory show` | 현재 세션의 메모리 내용을 표시합니다. |
| `/memory refresh` | GEMINI.md 파일을 다시 불러옵니다. |
| `/privacy` | 개인정보 수집 및 처리 관련 내용을 표시합니다. |
| `/restore <checkpoint_id>` | 이전 체크포인트에서 파일을 복원합니다. (옵션 필요) |
| `/stats` | 현재 세션의 통계 정보 (예: 토큰 사용량)를 표시합니다. |
| `/theme` | CLI 테마를 변경합니다. |
| `/tools desc` | 활성화된 도구들의 설명을 표시합니다. |
| `/tools nodesc` | 설명 없이 도구 이름만 나열합니다. |
| `/quit` 또는 `/exit` | Gemini CLI를 종료합니다. |

📂 **At (`@`) 명령어**

| 명령어 | 설명 |
| --- | --- |
| `@<file or directory path>` | 해당 파일 또는 디렉토리의 내용을 프롬프트에 삽입합니다. |
| `@` (단독 사용) | `@` 문자 자체를 그대로 프롬프트에 전달합니다. |

> ✅ .gitignore에 명시된 파일, 이진 파일, 대용량 파일은 자동 제외됩니다.
> 

🖥️ **느낌표 (`!`) 명령어**

| 명령어 | 설명 |
| --- | --- |
| `!<shell_command>` | 터미널 셸 명령어를 실행합니다. 예: `!ls -la`, `!git status` |
| `!` (단독 입력) | **셸 모드**로 전환 또는 해제합니다. 모든 입력이 셸 명령으로 처리됩니다. |

## 📝 참고 사항

- 일부 명령어는 서브 명령이 있는 구조(`/chat`, `/memory`, `/mcp`, `/tools`)이므로 하위 옵션과 함께 사용해야 합니다.
- `/restore`는 체크포인트 저장 옵션이 활성화되어 있어야 작동합니다.
- `/extensions`, `/privacy`는 GitHub 기준으로 공식 명령어에 포함되어 있습니다.
