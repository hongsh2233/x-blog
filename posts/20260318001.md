## App Router란?

Next.js 13부터 도입된 **App Router**는 기존 Pages Router와는 완전히 다른 아키텍처를 제공합니다. `app/` 디렉토리를 기반으로 하며, React 18의 서버 컴포넌트(Server Components)를 기본으로 활용합니다.

## 서버 컴포넌트 vs 클라이언트 컴포넌트

App Router의 핵심은 서버와 클라이언트 컴포넌트의 구분입니다.

```tsx
// 서버 컴포넌트 (기본값) - 서버에서만 실행됨
// app/page.tsx
async function HomePage() {
  const data = await fetch('https://api.example.com/posts');
  const posts = await data.json();

  return (
    <main>
      {posts.map((post: Post) => (
        <PostCard key={post.id} post={post} />
      ))}
    </main>
  );
}

export default HomePage;
```

```tsx
// 클라이언트 컴포넌트 - 'use client' 선언 필수
'use client';

import { useState } from 'react';

function LikeButton({ initialCount }: { initialCount: number }) {
  const [count, setCount] = useState(initialCount);

  return (
    <button onClick={() => setCount(c => c + 1)}>
      ❤️ {count}
    </button>
  );
}
```

## 레이아웃 중첩(Nested Layouts)

App Router의 강력한 기능 중 하나는 레이아웃 중첩입니다. 각 폴더에 `layout.tsx`를 두면 해당 경로와 하위 경로에서 공유 UI를 자동으로 적용할 수 있습니다.

```
app/
├── layout.tsx        ← 전체 공통 레이아웃 (nav, footer)
├── page.tsx          ← 홈 페이지
└── blog/
    ├── layout.tsx    ← 블로그 전용 레이아웃 (사이드바 등)
    └── [slug]/
        └── page.tsx  ← 개별 포스트 페이지
```

## 동적 라우팅과 generateStaticParams

정적 빌드 시 동적 경로를 미리 생성하려면 `generateStaticParams`를 사용합니다.

```tsx
// app/posts/[slug]/page.tsx
export async function generateStaticParams() {
  const posts = getAllPosts();
  return posts.map((post) => ({
    slug: post.slug,
  }));
}

export default function PostPage({ params }: { params: { slug: string } }) {
  const post = getPostBySlug(params.slug);
  return <article dangerouslySetInnerHTML={{ __html: post.content }} />;
}
```

## 정리

App Router는 초기 학습 곡선이 있지만, 서버 컴포넌트를 통한 성능 최적화와 직관적인 파일 기반 라우팅이 장점입니다. 특히 데이터 페칭을 컴포넌트 수준에서 처리할 수 있어 코드 구조가 훨씬 깔끔해집니다.
