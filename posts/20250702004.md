**바이브  코딩은 AI 코딩이 발전하면서 파생된 말로 자연어(한글) 로 코딩** 하는 것을 말해요!

“사용자 이름 입력받고, 로그인 버튼 누르면 홈 화면 보여줘” 같은 문장만 써도 AI가 알아서 코드로 만들어주는 거죠. 아예 **기존 코드 없이 말로만 코딩**을 하는 거에요!!!

옛날엔 **기계어 → 저급어 → 고급어 단계로 발전했잖아요? 이제는 자연어가 ‘최고급 언어’** 가 된 셈!

AI가 코드를 자동 생성하면서 점점 더 자연스러운 대화형 코딩 환경이 만들어지고 있어요.

![컴퓨터 언어의 진화 과정…ChatGPT 가 이미지 만들어주기 귀찮나봐요](/images/posts/20250702004/02.png)

컴퓨터 언어의 진화 과정…ChatGPT 가 이미지 만들어주기 귀찮나봐요

그렇다면! **왜 이렇게 HOT**🔥 ****한 것일까요?

1. **코딩 진입장벽↓** : 프로그래밍 언어 문법 몰라도 자연어로 쉽게 시작 가능
2. **생산성↑** : 반복적인 코드 작성 시간 확 줄임
3. **협업도 쉬워짐** : 비개발자도 요구사항을 자연어로 전달 가능

간단하게 위와 같은 이유들로 인해서 바이브 코딩이 각광 받는 시대가 오고 있어요.

더 이상 개발자가 아니여도 코딩을 할 수 있는 시대가 다가오고 있다는 것이죠!

하.지.만  **문제가 없는 건 아니에요… 바이브 코딩 개발자와 기존 개발자의 주장** 을 보면 알 수 있어요.

## **바이브 코딩 개발자 vs 기존 개발자**

![누가 누가 더 빨리 만드는지 점심 커피 내기중인 바이브 코딩 개발자와 기존 개발자의 모~습](/images/posts/20250702004/03.png)

누가 누가 더 빨리 만드는지 점심 커피 내기중인 바이브 코딩 개발자와 기존 개발자의 모~습

바이브 코딩으로 개발하고 있는 사람들은 지금까지 언어가 발전해온 과정처럼 “이제는 자연어로 코딩하는 시대가 왔을 뿐” 이라고 해요.

반대로 **기존 개발자**는  **코드의 기본을 모르면 유지보수나 “협업이 불가능 할 것이다”**  라고 합니다.

많은 현업 개발자들의 의견이 기본적인 코드와 구조, 컴퓨터의 이해가 필요하다고 말해요!

같은 주제로 글을 써도 글의 구조, 표현, 비유 등 공부를 한 사람과 AI가 작성한 글에서 미묘한 차이가 느껴지듯 디테일하고 세부적인, **사람만이 할 수 있는 일이 있다고 생각** 하거든요!

또  **프로젝트가 방대할수록 AI의 한계가 분명** 하고 **코드의 구조 요소 외에 사람만이 가질 수 있는 회사의 성격이나 관습 등 외부적인 요소가 작용**하기 때문에 더욱이 AI 만으로는 대체가 어려울 것이라고 얘기해요.

하지만  **바이브 코딩 개발자** 들은 이 또한 **기술 발전**으로 인해서 **빠른 시일 내에 해결 될 문제**라 말해요.

또 **만들어진 코드에 문제가 생겨도 “3초면 새로운 코딩을 뽑아내는데 무엇이 문제야??”** 라고 하더라구요!

## **바이브 코딩… 앞으로 어떻게 될까?**

AI가 코딩 생산성을 높여주지만, 뼈대가 되는 프로그래밍 기본 지식과 로직 설계 능력은 여전히 필수!

**AI가 발전할수록 튼튼한 기초 지식과 코딩 지식이 빛을 바랄 것**이라고 많은 사람들이 얘기해요.

**자연어 코딩은 ‘보조 도구’**로 자리 잡고, **개발자는 더 고차원적인 문제 해결과 창의적 설계에 집중**하게 될 것 같아요.

결국 **개발자는 ‘AI 활용 능력 + 기본기’ 두 마리 토끼를 잡아야 하는 시대**가 오겠죠!

## **MCP는 뭔데? Many Chicken Please…?** 😫

세상이 너무 빨리 발전해요… 매일 새로운 단어가 나오는데 무슨 줄임말인 것 같은데 😭

네 제 이야기입니다.. 뭔가 싶었어요. 하지만 **알고 나면 어렵지 않.아.요!**

**MCP는 Model Context Protocol 의 줄임말로 LLM(AI) 가 외부 데이터와 시스템을 더 효과적으로 사용할 수 있도록 만든 프로토콜** 이에요! 이것도 무슨 말인지 어렵죠..?

간단한 예로 **핸드폰 충전기가 여러 종류라 불편했는데 지금은 c타입으로 통합**된 것처럼, **AI와 앱, 데이터가 연결될 때도 통일된 규칙이 필요해 기준**을 만든 거에요!

## **MCP가 앞으로 넘어야 할** 🏔️

MCP가 AI와 다양한 앱, 서비스 간 연결을 원활하게 해주지만, 아직 **모든 서비스가 MCP를 지원하지는 않아요. 완전한 통합**을 위해서는 **더 많은 기업과 플랫폼이 MCP 표준을 채택하고 호환성을 맞춰야 하는 과제**가 있어요! 참고로 인터넷 프로토콜이 통합되는데는 약 25년이라는 세월이 걸렸답니다 👻

또 AI가 여러 앱과 데이터를 자유롭게 연결하고 활용하는 만큼, **사용자 개인정보와 데이터 보안 문제**도 중요해요. 그래서 MCP를 사용할 때는 **각 앱 간 데이터 전송과 처리 과정에서 안전한 암호화와 권한 관리**가 반드시 필요해요!

마지막으로 AI가 여러 프로그램을 연동해 복잡한 작업을 수행할 수 있지만, **명령이 너무 복잡하거나 모호하면 오작동**할 수도 있어요. 많은 기업과 플랫폼이 참여해서 하루 빨리 사용자 명령어 설계와 AI의 이해도를 높이는 연구와 개선이 되었으면 좋겠어요!

## **A2A가 MCP 와 손잡고 앞으로 넘어야 할** 🏔️

**상호운용성(Interoperability)** **문제**가 있어요.

서로 다른 AI 에이전트들이 원활하게 소통하려면 공통의 데이터 형식과 통신 규약이 필요한데 각 AI 회사마다 사용하는 기술과 구조가 달라 완전한 상호운용성을 확보하는 데 시간이 걸릴거에요.

또 **신뢰성과 보안 문제**로 AI끼리 데이터를 주고받으며 협업하는 과정에서 악의적 행위나 데이터 위변조 위험이 존재해요. 에이전트 간 신뢰 구축, 인증, 권한 관리, 데이터 암호화 등 보안 체계가 반드시 강화되어야 할 거에요.

![오늘의 점심내기 - AI가 스스로 흑화하는게 빠를까, 해커들이 흑화시키는게 빠를까](/images/posts/20250702004/08.jpg)

**복잡한 협업 시나리오 관리의 어려움**도 있어요.

여러 AI가 동시에 협력할 때 역할 분담, 충돌 해결, 작업 우선순위 조정 등 복잡한 관리가 필요한데 이를 위한 표준화된 프로토콜과 AI 내부의 의사결정 메커니즘 개발이 필수적이에요.

마지막으로 **책임 소재와 윤리 문제**가 있어요.

AI 에이전트 간 협업으로 발생하는 결과에 대해 누가 책임을 지는지 명확하지 않을 수 있는데 특히 자율적 의사결정이 많아질수록 복잡해지기 때문에 윤리적 가이드라인과 법적 규제도 함께 마련되어야해요.

## **A2A가 가져올** 🍯**같은 미래**
